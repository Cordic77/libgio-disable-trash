Globally disable GNOMEʼs Trash in Debian-based distributions

GNOMEʼs built-in Trash has one particularly annoying feature: every time one deletes a file on a mounted drive, a ".Trash-<UID>" folder is created. This happens regardless of whether the storage device in question is formatted with a Linux filesystem, or not (e.g. USB flash drives or Windows shared folders).

If you are like me and want the Trash gone for good, here is a solution that will disable this "feature" globally, system-wide for everyone:


  1. BEWARE: this guide proposes to replace GNOMEʼs libgio with a custom-compiled version which re-routes all calls of g_file_trash() to its pendant g_file_delete().
    Modifying system libraries is not without risks – I cannot accept any responsibility for any loss, disruption or damage to your data or computer system which may occur as a result of following the steps provided in this guide!
    Also, the following procedure will need to be repeated every time a system update causes a new libglib to be installed:

    $ sudo apt full-upgrade
    Reading package lists... Done
    Building dependency tree
    Reading state information... Done
    Calculating upgrade... Done
    The following packages will be upgraded:
    libglib2.0-0


  2. Before getting started, make sure the latest updates are installed:

    $ sudo apt full-upgrade

    Also, it would be wise to get answers to some basic questions first:
    • Which package contains GIO?

      $ apt-file search -x 'libgio-.*?\.so'
      libglib2.0-0t64: /usr/lib/x86_64-linux-gnu/libgio-2.0.so.0
      libglib2.0-0t64: /usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4

      See also: Glib
    • Where are these files located?

      $ find /usr/lib/ -regextype posix-extended -regex '.*/libgio-2\.0\.so\.0(\.[0-9]+){1,2}'
      /usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4

      Note: all commands given in the remainder of this guide will refer to this specific version of libgio – on other systems, some adjustments will be necessary!

  3. Download source package for glib2.0:

    $ sudo apt source glib2.0
    $ ls -l
    drwxr-xr-x 8 root root 4096    2026-06-21 11:03 glib2.0-2.84.4
    -rw-r--r-- 1 root root 153388  2026-06-21 11:03 glib2.0_2.84.4-3~deb13u3.debian.tar.xz
    -rw-r--r-- 1 root root 4883    2026-06-21 11:03 glib2.0_2.84.4-3~deb13u3.dsc
    -rw-r--r-- 1 root root 5618200 2026-06-21 11:03 glib2.0_2.84.4.orig.tar.xz
    -rw-r--r-- 1 root root 660708  2026-06-21 11:03 glib2.0_2.84.4.orig-unicode-data.tar.xz


  4. Install build dependencies for glib2.0:

    $ sudo apt build-dep glib2.0


  5. Extract glib2.0_2.84.4.orig.tar.xz into a newly created subdirectory ./extr:

    $ mkdir -p extr && tar xf glib2.0_2.84.4.orig.tar.xz -C extr/ && cd extr


  6. Open the file "glib-2.84.4/gio/gfile.c" in an editor of your choice, locate the function g_file_trash() and replace its body with a
    single return (g_file_delete (...)) statement, as shown below (any existing code code should be commented out with /* ... */):

    $ sudo gedit glib2.0-2.84.4/gio/gfile.c

gboolean
g_file_trash (GFile         *file,
              GCancellable  *cancellable,
              GError       **error)
{
  /* (Unconditionally) bypass the trash: */
  return (g_file_delete (file, cancellable, error));

/*// Existing code commented out:
  GFileIface *iface;

  g_return_val_if_fail (G_IS_FILE (file), FALSE);

  if (g_cancellable_set_error_if_cancelled (cancellable, error))
    return FALSE;

  iface = G_FILE_GET_IFACE (file);

  if (iface->trash == NULL)
    {
      g_set_error_literal (error,
                          G_IO_ERROR, G_IO_ERROR_NOT_SUPPORTED,
                          _("Trash not supported"));
      return FALSE;
    }
/usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4
  return (* iface->trash) (file, cancellable, error);
*/
}
    • A comparison of this (modified) gfile.c with the one in the glib2.0-2.84.4 directory downloaded in step 2 should now reveal a single change:

      $ diff -b glib-2.84.4/gio/gfile.c ../glib2.0-2.84.4/gio/gfile.c
      4779,4782d4778
      < /* (Unconditionally) bypass the trash: */
      < return (g_file_delete (file, cancellable, error));<
      < /*// Existing code commented out:
      4801d4796
      < */

    • If the comparison in the previous step checks out, weʼre ready to proceed:

      # Copy the modified gfile.c over the one extracted in step 2:
      $ sudo cp -a glib-2.84.4/gio/gfile.c ../glib2.0-2.84.4/gio/gfile.c

      # Repack the glib2.0_2.84.4.orig.tar.xz archive:
      $ sudo tar cJf ../glib2.0_2.84.4.orig.tar.xz . && cd ..
      $ ls -l glib2.0_2.84.4.orig.tar.xz
      -rw-r--r-- 1 root root 5697064 glib2.0_2.84.4.orig.tar.xz

      # Remove the temporary extr directory:
      sudo rm -rf extr/


  1. Change into the glib2.0-2.84.4 directory downloaded in step 2, log in as the root user and run dpkg-buildpackage:

    $ cd glib2.0-2.84.4/
    $ sudo su root
    # dpkg-buildpackage

    Notes:
    1. In some cases, one of GLibʼs automated tests may "hang", thus preventing dpkg-buildpackage from completing the build:

      PASS: gsettings 23 /gsettings/no-read-binding
      ^Cmake[9]: *** Deleting file 'gsettings.log'
      Makefile:4163: recipe for target 'gsettings.log' failed
      make[9]: *** [gsettings.log] Interrupt

    2. If one is willing to skip the test suite, itʼs possible to avoid this by invoking dpkg-buildpackage with the nocheck option (after having aborted the hanging test with <Ctrl-C>):

      # DEB_BUILD_OPTIONS=nocheck dpkg-buildpackage -us -uc -b


  2. A successful run of dpkg-buildpackage will result in a set of newly created .DEB archives, which could be installed with dpkg:

    # exit
    $ ls -l ../libglib2.0-0*.deb
    -rw-r--r-- 1 root root 1517380 ../libglib2.0-0t64_2.84.4-3~deb13u3_amd64.deb
    -rw-r--r-- 1 root root 4583748 ../libglib2.0-0t64-dbgsym_2.84.4-3~deb13u3_amd64.deb


  3. For our purposes, however, it is better to simply replace the systems libgio-2.0.so.0.8400.4 with the newly compiled one (after making a backup copy):
    # Backup current 'libgio':

    $ sudo cp -a /usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4{,-installed}

    After that itʼs necessary to reboot the system into the recovery mode, as the last step wonʼt work while the GUI is running (it's also possible to boot into textmode only by temporarily adding the »systemd.unit=multi-user.target« kernel boot-parameter to the systemʼs default GRUB entry):
    # Install libgioʼs replacement:

    root@deb:~# cd /home/<your user>/libgio/glib2.0-2.84.4/
    root@deb:/.../glib2.0-2.84.4# find ./debian/libglib2.0-0t64/usr/lib/ -name 'libgio-2.0.so.0.*' -ls
    -rw-r--r-- 1 root root 2051320 ./debian/libglib2.0-0t64/usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4
    root@deb:/.../glib2.0-2.84.4# cp -a ./debian/libglib2.0-0t64/usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4 /usr/lib/x86_64-linux-gnu/libgio-2.0.so.0.8400.4

    Note:
    Please also make sure that all libgio-2.0.so* symlinks under /usr/lib/x86_64-linux-gnu actually point to this updated libgio-2.0.so.0.8400.4!


  4. Reboot your system - the Trash will now no longer be used!
